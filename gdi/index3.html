Block elements vs inline elements

Block elements vs inline elements

Block: p, h1, h2, ul, li, most things

Inline: images, links, em, strong

The DIV element

Used to group elements to format them with CSS. Is not a semantic HTML element.

The SPAN element

Used to apply styles to text inline. Also not a semantic HTML element.

Display inline vs block vs inline-block

Read this blog post

As you can see in the example, display: inline-block causes whitespace in between elements. Float left does not.

Image file formats

Blog post: PNG vs GIF vs JPG

JPG: best for large background images. Cannot hold transparency.

PNG: best for most use cases. Can hold transparency.

GIF: good for animations.

SVG: made up of only vectors!

Property shorthands

Padding: 10px 25px 30px 5px;

Border: 1px solid red;

Margin: 2em auto;

Font: font-style font-variant font-weight font-size/line-height font-family;

Font: italic small-caps normal 13px/150% Arial, Helvetica, sans-serif;



margin: 0 = all sides of the margin is zero

padding: 10px 20px = top and bottom are 10px. Left and right are 20px.

border-width: 5px 8px 10px = top is 5px, left and right is 8px, bottom is 10px.

padding: 3px 5px 7px 9px = top is 3px, right is 5px, bottom is 7px and left is 9px. Mnemonic for TRBL is TRouBLe
